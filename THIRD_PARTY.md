# Third-Party Sources

What external material this repository uses, where each piece came
from (a public URL and a full revision, version, or content hash, as
applicable), under what terms, and exactly what was taken. The
project's own code is Apache-2.0 (see `LICENSE`).

## Position

Stated explicitly rather than assumed:

- **No third-party source code is copied, adapted, translated,
  compiled, or linked.** External sources contributed *numeric
  values* only — astronomical positions, published worked-example
  sextant readings and answers, and correction-table entries —
  transcribed into integer test fixtures by the one-time offline
  tools in `tools/` (which document every source and revision, and
  never run at build or test time).
- **Primary sources are preferred and named.** Wherever a value has
  a government or agency primary source it is anchored to that
  source here: United States government works (the USNO Air Almanac,
  Bowditch) are public domain; the Nautical Almanac's tabulated
  astronomical data are joint HMNAO/USNO computed tabulations; JPL
  DE421 is publicly released NASA/JPL data. GitHub repositories
  listed below served as collection and finding aids for those
  printed values, and as independent computed implementations for
  cross-checks — not as the origin of the data.
- Individual measured or computed values and rights in a *compiled
  database* are separate questions, and jurisdictions treat them
  differently (US compilation copyright; the EU database directive's
  extraction right). The selections here are deliberately small —
  spot-check rows and single worked examples, an insubstantial
  fraction of any source, re-expressed in this project's own
  structure — and every one is attributed and pinned below
  regardless.
- The collection repositories carry their own licenses (two
  GPL-3.0, one MIT with an added attribution clause, noted per
  entry). Those terms attach to their software, none of which is
  used here. The ESA Hipparcos catalogue's terms are stated in its
  entry. This is flagged here rather than assumed silently.
- If you hold rights in a cited source and see it differently, open
  an issue: every transcribed row is pinned and can be re-traced
  directly to a primary source, regenerated from one, or removed.

## Committed fixtures

### `ephemeris_reference.h` (generated by `tools/make_ephemeris_reference.py`)

Computed truth vectors for the `--ephemeris-check` gate: apparent
Sun/Moon/star places computed with **Skyfield 1.54** (MIT) from
**JPL DE421** and, for the stars, the **ESA Hipparcos catalogue**
(Credit: ESA — see "Regeneration-time data" below). Computed output,
not copied content; the header banner records the library version
and input-file SHA-256 hashes.

### `external_reference.h` (generated by `tools/make_external_reference.py`)

Rows transcribed from printed almanacs and the public test suites
that quote them (about 1,250 rows in total), consumed by the
`--external-check` gate:

- Daily pages of the Paradise Cay 2021 Nautical Almanac reprint, the
  2002 Nautical Almanac, the USNO **Air Almanac 2023** (US
  government work, public domain), and the EZ Celestial Almanac 2023
  — collected via the `daily-pages-*.txt` files of
  <https://github.com/quantenschaum/nautical_almanac>
  (repo license GPL-3.0), revision
  `799a9c9667bad428f450a8559063010cbabc871e`.
- Nautical Almanac 2001–2025 Sun/Moon/star rows from the "Nautical
  Almanac" reference column of `test/altitude_tests.cpp`
  (`SUNMOON_SIGHTS`/`STAR_SIGHTS`) in
  <https://github.com/rgleason/celestial_navigation_pi>
  (repo license GPL-3.0), revision
  `e442c28ae0aeea98429391a143592c4c96f1d3a5`.
- Nautical Almanac refraction and dip table entries via
  <https://github.com/alejandrozarco/celestial-navigator> (MIT),
  revision `c09c04dcb950ed1ae27f08dd1c2992c48b85f990`.
- Nautical Almanac GHA-Aries wrap values and the worked example from
  Bowditch, *American Practical Navigator* (2019 ed., p. 315; US
  government work, public domain) as quoted in `test_almanac.py` of
  <https://github.com/DaniilGalahov/celnav> (MIT License with
  Attribution Clause — credit to Daniil Galakhov; acknowledged here,
  though no part of that software is included in this repository),
  revision `54e81da6fd29c7c8fdd16e410a00c5747e624dde`.

The Nautical Almanac is a joint HMNAO/USNO publication; only small
selections of its tabulated astronomical values are transcribed, as
spot checks, for validation.

### `cross_reference.h` (generated by `tools/make_cross_reference.py`)

Computed cross-implementation rows for the `--cross-check` gate:
almanac places computed with **PyEphem 4.2.1**
(<https://github.com/brandon-rhodes/pyephem>, MIT; libastro © Elwood
Downey, Python package © Brandon Rhodes — an implementation lineage
independent of Skyfield/JPL), plus intercept-geometry cases checked
against <https://github.com/alinnman/celestial-navigation> (MIT),
revision `b3ab049a291ac4da0a0431e798a2125f549bcd8d`. Computed
output; the generator enforces its version and revision pins.

### `sight_scenarios.h` (generated by `tools/make_sight_scenarios.py`)

Whole-pipeline scenarios for the `--scenario-check` gate:

- Truth positions and body places computed with Skyfield 1.54 /
  DE421 / the ESA Hipparcos catalogue (Credit: ESA), as above.
- Two published worked fixes (raw sextant readings and the published
  answers) transcribed from
  <https://github.com/alinnman/celestial-navigation> (MIT), revision
  `b3ab049a291ac4da0a0431e798a2125f549bcd8d`.
- About ten Sun intercept rows (each traceable to the Nautical
  Almanac) from `test/altitude_tests.cpp` (`INTERCEPT_SIGHTS`) in
  <https://github.com/rgleason/celestial_navigation_pi> (GPL-3.0),
  revision `e442c28ae0aeea98429391a143592c4c96f1d3a5`.

### Star catalog (in `astro_nav.c`)

J2000 places for 18 bright navigational stars (17 of the almanac's
57 selected stars, plus Polaris), committed as integer unit vectors.
Bright-star positions at navigational precision are standard,
multiply-published values — the same stars appear in the Nautical
Almanac's and the Air Almanac's star pages and in Bowditch (US
government works among them), and the in-source comments carry the
classical SHA/declination for cross-checking against printed almanac
star pages. The committed values are Hipparcos-era (Credit: ESA for
the catalogue heritage) and are gated against Skyfield + Hipparcos
apparent places and printed almanac pages by `--ephemeris-check` and
`--external-check` rather than trusted.

### Built-in self-test rows (in `sight_reduction.c`)

A few sight-reduction truth-table case geometries (the
latitude/declination/LHA selections, and the cardinal and singular
edge cases) mirror the test suite of
<https://github.com/alejandrozarco/celestial-navigator> (MIT),
revision `c09c04dcb950ed1ae27f08dd1c2992c48b85f990`, and classic
Pub. 229 sight-reduction-table entries (US government publication,
public domain). The expected values themselves are computed by this
repository's own offline IEEE-double oracle evaluating the published
spherical-trig formula, not transcribed.

### `fp_math.h`

Vendored from the same author's Apache-2.0 repository
<https://github.com/nmicic/int-llm>, revision
`8b16a00bb9126d2a99db9ad7683c3c064b9eb7fc`, with intentional
divergences documented in its banner. Same author and license;
listed for completeness.

## Regeneration-time data (not committed)

Downloaded to `tools/skyfield-cache/` only when regenerating
fixtures; never present in, or redistributed by, this repository:

- `de421.bsp` — JPL planetary ephemeris DE421, publicly released
  NASA/JPL data
  (<https://ssd.jpl.nasa.gov/ftp/eph/planets/bsp/de421.bsp>),
  SHA-256
  `a20a7139da04cbc462454634918e9a9ca69127044e2cc9d4f9c16e238d2deedc`.
- `hip_main.dat` — The Hipparcos Catalogue (ESA SP-1200, 1997).
  ESA distributes the Hipparcos and Tycho catalogues under
  **CC BY-NC 3.0 IGO** with the required credit line "**Credit:
  ESA**" (<https://www.cosmos.esa.int/web/hipparcos/catalogues>).
  Credit: ESA. Used here only as a validation-time input: the
  offline tools read it to compute apparent-place truth values that
  the committed gates then check the library against. The catalogue
  file itself is never committed. SHA-256
  `58ceabb104d647160d9437ce6e513a02a036bb4ad9f8879a5a22fd52943616e0`.
- Python packages pinned by the generators: `skyfield==1.54` (MIT),
  `ephem==4.2.1` (MIT), plus numpy/pandas as recorded in each tool.

## Algorithmic sources

The ephemeris core implements truncated series and reduction methods
from the standard astronomical literature (Meeus, *Astronomical
Algorithms*, and the almanac correction-table definitions in
Bowditch and the Nautical Almanac) — published formulas and methods,
independently implemented in integer arithmetic; no code from any
reference implementation. JPL DE421 and Skyfield serve as validation
oracles only; no DE421 data is embedded.
